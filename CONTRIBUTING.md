- クラッシュログやスクリーンショット（可能な場合）

## ソースコードの構成

```text
AviQtl/
├── core/           # アプリケーション基盤ロジック（FFmpeg、ECS定義、シリアライズ）
├── engine/         # 信号処理・レンダリング（オーディオミキサー、計算ロジック）
├── ui/             # ユーザーインターフェース
│   ├── include/    # C++ サービス・モデル
│   └── qml/        # Qt Quick (QML/JS) 定義
│       ├── effects/ # エフェクトロジック & GLSL
│       └── common/  # 共通コンポーネント
├── scripting/      # スクリプトエンジン (LuaJIT FFI)
├── i18n/           # 翻訳ファイル (.ts)
├── assets/         # 静的リソース（アイコン、スプラッシュ）
├── BUILD.py        # クロスプラットフォーム・ビルドスクリプト
└── check.py        # 静的解析・フォーマット一括実行ツール
```

### 設計指針

- **データ指向型 (ECS):** 高いパフォーマンスとキャッシュ効率を優先します。
- **モダン C++:** C++23 の機能を活用し、安全で効率的なコードを目指します。
- **クロスプラットフォーム:** 特定の OS API への依存を避け、抽象化レイヤーを介するようにします。

### 技術スタック

| カテゴリ | 採用技術 |
|----------|----------|
| 言語 | C++23, QML, JavaScript, LuaJIT, GLSL (C++標準に準拠) |
| フレームワーク | Qt 6 (Qt Quick, Multimedia, ShaderTools) |
| グラフィックス | Vulkan |
| 音声 | Carla (VST3 / LV2 / LADSPA host) |
| ビルドシステム | CMake, Ninja (x86_64 / ARM64 対応) |

## 開発の進め方

### 1. 準備

- [Codeberg のリポジトリ](https://codeberg.org/taisho-guy/AviQtl)をフォークします。
- フォークしたリポジトリをローカルにクローンします。

### 2. 環境構築

- `README.md` の「ビルド手順」に従い、依存関係をインストールします。
- `BUILD.py` を使ってビルドが通ることを確認してください。

### 3. ブランチの作成

変更内容に合わせて、以下の命名規則でブランチを作成してください。

```bash
git switch -c fix/bug-description   # バグ修正
git switch -c feat/feature-name     # 新機能追加
```

### 4. コミットとプッシュ

- 変更内容が伝わるコミットメッセージを記述してください。
- 変更を自分のフォークへプッシュします。

### 5. プルリクエストの作成

- Codeberg 上でオリジナルリポジトリに対して PR を作成します。
- PR の説明には、変更の理由と修正内容を簡潔に記載してください。
